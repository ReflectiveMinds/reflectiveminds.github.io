The Toolkit

RPG Combat Toolkit (TBCS)

A complete, production-ready framework for grid-based, turn-based combat in Unity — NodeGraph stat formulas, flexible action points, multi-tile actors, and ready-to-play UGUI combat scenes.

🔧
Complete Combat System, Day One

Movement, targeting, attacks, abilities, statuses, AP, and overlays — all wired together in ready-to-play scenes. Every system implements an interface, so nothing is locked in and everything is replaceable.

🗺️
One Codebase. Two Worlds.

One toggle switches between top-down 2D and isometric 3D. Visualizers, overlays, movement, and multi-tile actors (1×1 → 3×3) all adapt automatically — no duplicate code.

🎯
Polished Targeting, Out of the Box

Pluggable evaluators for any shape, range, or filter — polished zone previews, line arrows, and instant valid/invalid feedback. From single strikes to large AoE spells.

💾
Save Mid-Combat. Resume Anywhere.

Support single or multiple AP pools. A snapshot layer captures the full combat state — actors, turn order, statuses, inventory — so players can save and resume mid-combat safely.

⚙️
Your Rules. No Core Edits.

Define action costs, turn economy, cover rules, opportunity attacks, and stat interactions without touching core code. Tune for classic CRPGs, tactics games, or your own system.

Under the hood

Technical highlights

Built for engineers who care about architecture.

Service-driven architecture

Core systems (actors, stats, abilities, overlays, persistence, animation) communicate through interfaces and a service registry.

2D/3D dual-mode support

One settings object drives 2D vs 3D — movement, overlays, equipment, and visualizers all respect the chosen mode.

Modular abilities pipeline

Abilities defined as pipelines: targeting, validation, execution, effects, and visuals — easy to extend or replace each stage.

GPU-instanced overlays

Attack and range previews use instanced meshes and depth-aware shaders for clean, performant visuals.

Save-ready combat loop

Combat snapshots capture turn order, actors, statuses, inventory & equipment for safe mid-combat restores.

Extensible by design

Everything built to be swapped: stat systems, health, animation drivers, inventory, AI, and beyond.

FAQ

Common questions

Does it work with my existing save system?

Yes. The persistence layer uses plain interfaces — swap in your own serializer without touching any core code.

Is it compatible with DOTween or other animation tools?

Yes. The animation driver is a swappable interface. Plug in DOTween, LeanTween, or your own solution freely.

Can I use this for a pixel-art 2D game?

Absolutely. TBCS is renderer-agnostic — visuals are fully decoupled from the combat logic. Use any art style.

Does it support multiplayer?

Not out of the box, but the architecture separates state from presentation — a solid foundation if you plan to add netcode later.

Which Unity versions are supported?

Unity 2021.3 LTS and above, including Unity 6. Both URP and Built-in render pipelines are supported.

When will it be on the Unity Asset Store?

We're targeting a 2026 launch. Join the notify list above to get early-bird pricing the moment it goes live.
