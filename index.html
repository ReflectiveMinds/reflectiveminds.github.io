Technical highlights

Service-driven architecture

Core systems (actors, stats, abilities, overlays, persistence, animation) talk through interfaces and a service registry.

2D/3D dual-mode support

A single settings object drives 2D vs 3D behavior ‚Äî movement, overlays, equipment, and visualizers respect the chosen mode.

Modular abilities

Abilities are defined as modular pipelines: targeting, validation, execution modules, effects, and visuals ‚Äî easy to extend or replace.

GPU-instanced overlays

Attack and range previews use instanced meshes and depth-aware shaders for clean, non-blocky visuals.

Save-ready combat loop

Combat snapshot adapters capture turn order, actors, statuses, inventory & equipment, and more for safe mid-combat restores.

Extensible by design

Everything is built to be swapped: stat systems, health implementations, animation drivers, inventory, AI, and beyond.
